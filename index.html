Azure VM Node.js Deployment with CI/CD

Updating the Node.js application on Azure virtual machines was manual and slow. Each change required logging into the server and setting things up by hand, which made deployments inconsistent and time‑consuming.

Automated the deployment by using Terraform to provision the Azure VM and networking, then set up a CI/CD pipeline in Azure DevOps. The pipeline pulled the latest code from the repo and deployed the Node.js application with NGINX as a reverse proxy. This removed the need to log in and update the server manually.

Updates became fast, reliable, and consistent. Every change to the codebase triggered an automated rollout, reducing downtime and human error. The project showed strong skills in combining infrastructure automation with application deployment, making delivery smoother and more professional.
